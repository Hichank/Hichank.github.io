webpack优化配置

production模式打包自带的优化

  • tree shaking

tree shaking 是一个术语, 通常用于打包时移除 JavaScript 中未引用的代码 (dead-code), 它依赖于 ES6模块系统中的 import 和 export 静态结构特性。

开发时引入一个模块后, 如果只使用其中一个功能, 上线打包时只会把用到的功能打包进文件, 其他没用到的功能不会打包进来, 可以实现最基础的优化

  • scope hoisting

    Scope hoisting的作用是将模块之间的关系进行结果推测, 可以让 webpack打包出来的代码文件更小, 运行更快

    scope hoisting 的实现原理就是在不造成代码冗余的前提下分析模块之间的依赖关系把零碎的模块合并到一个函数中, 由于 scope hoisting需要分析出模块之间的依赖关系 因此源码必须采用 ES6模块化语法

  • code compression

    所有代码使用UglifyJsPlugin插件进行压缩 混淆

css优化

  • 将css提取到独立文件中

mini-css-extract-plugin 是用于将 css 提取为独立文件的插件, 对每个包含 css 的 .js 文件都会创建一个 .css 文件, 支持按需加载css和sourceMap

此插件只能在 webpack4.0以上使用, 优点:

  • 异步加载
  • 不重复编译 性能好
  • 易用
  • 只针对css

使用 postcss 需要用到 postcss-loader 和 autoprefixer 插件

开启css压缩

需要使用 optimize-css-assets-webpack-plugin 插件来完成css压缩

由于配置css压缩时会覆盖webpack默认的优化配置, 导致 js代码无法压缩, 所以还需手动把 js代码压缩插件导入进配置文件中, 这里我们建议使用 : terser-webpack-plugin

  1. 安装

  2. 在 webpack.prod.js 中配置 (css代码压缩适用于在production模式下)

    Tips: webpack4.0+默认采用的 js压缩插件为: uglifyjs-webpack-plugin , 在 mini-css-extract-plugin 上一个版本中也推荐使用该插件, 但最新的 v0.6中建议使用 terser-webpack-plugin 来完成 js代码的压缩, 具体原因未在 npm官网进行说明

js优化

Code splitting 是 webpack打包时用到的重要的优化特性之一, 此特性能够把代码分离到不同的 bundle文件中, 然后可以按需加载或者并行加载这些文件。代码分离可以用于获取更小的 bundle, 以及控制资源加载优先级, 如果使用合理, 会极大影响加载时间。

有三种常用的代码分离方法 :

  • 入口起点 (entry points) : 使用 entry 配置手动地分离代码
  • 防止重复 (prevent duplication) : 使用 SplitChunksPlugin 去重 然后分离 chunk
  • 动态导入 (dynamic imports) : 通过模块的内联函数调用来分离代码
手动配置多入口

  1. 在 webpack.config.js 配置文件中设置多个入口

  2. 这种方法打包完会存在一些问题 :

    • 如果入口 chunks 之间包含重复的模块, 那些重复模块都会被引用到各个 bundle中
    • 这种方法不灵活, 并且不能讲核心应用程序逻辑进行动态拆分代码

  1. 运行 npm run build 重新打包

动态导入

webpack4.0+ 默认是允许 import 语法动态导入的, 但是需要 babel 的插件支持, 最新版本的 babel 的插件包为 @babel/plugin-syntax-dynamic-import

提高构建性能

noParse

在引用一些第三方模块时, 例如 jQuery/bootstrap等, 我们知道这两个内部不会有相互依赖, 因为最终我们用到的只是一个单独文件, 而 webpack会去解析他们内部依赖关系, 我们需要阻止webpack的默认行为.

可以在 webpack配置文件中的 module 节点下加上 noParse , 并配置正则来确定不需要判断的依赖关系的模块

lgnorePlugin

在引入一些第三方模块时, 例如 moment, 内部会做国际化处理, 所以会包含很多语言包, 而语言包打包时会比较占空间, 如果我们项目只需要少数语言, 可以忽略掉其他的所有语言包, 然后按需引入语言包从而使得构建效率更快, 打包生成的体积更小

需要忽略第三方模块内部依赖的其他模块, 只需要做如下处理 :

  1. 安装并找到 moment依赖的语言包

  2. 使用 ignorePlugin 插件忽略其依赖

  3. 按需手动引入
