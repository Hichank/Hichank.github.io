  1. 数据可以直接渲染，对于搜索引擎（SEO）友好

第三方库的两种引入方式

可以通过 expose-loader 进行全局变量的注入, 同时也可以使用内置插件 webpack.ProvidePlugin 对每个模块的闭包空间注入变量, 自动加载模块, 而不必到处 import 或 require

  • expose-loader 将库引用到全局作用域

    1. 安装 (例: juqery)

    2. 在 webpack.config.js 中配置loader

      require.resolve 用来获取模块的绝对路径

  • webpack.ProvidePlugin 将库引用到所有模块

    ProvidePlugin插件是webpack是内置的

Development / Production不同配置文件打包

项目开发时一般需要用到两套配置文件, 用于开发阶段打包 (不压缩代码, 不优化代码, 增加开发效率) 和上线阶段打包 (压缩代码, 优化代码, 打包后直接上线使用)。

抽取三个配置文件

  • webpack.config.js
  • webpack.prod.js
  • webpack.dev.js

步骤如下

  1. 将开发环境和生产环境公用配置放入 webpack.config.js 中, 不同的配置各自放入 webpack.prod.js 或 webpack.dev.js 中 (例 : mode)

  2. 在 webpack.prod.js 或 webpack.dev.js 中使用 webpack-merge 将各自配置与公用配置合并后导出

    安装 npm i webpack-merge -D

  3. 将 package.json 中的脚本参数进行修改, 通过 --config 手动指定特定的配置文件

proxy代理

开发阶段往往会遇上 跨域 的问题, 但在上线后前端代码会和后端代码整合部署在 同一服务器 下, 并不会出现跨域的问题, 但是由于开发时用到 webpack-dev-server 所以一定会产生跨域的问题

目前跨域的主要解决方案有 :

  1. jsonp (淘汰)
  2. CORS
  3. http proxy

此处介绍的使用devServer解决跨域 原理是http proxy

将所有 ajax请求发送给devServer服务器, 再通过devServer服务器转发到数据接口服务器

因为跨域问题的起因是[浏览器的同源策略][https://developer.mozilla.org/en-US/docs/Web/Security/Same-origin_policy], 由于ajax请求是发送给devServer服务器, 所以不存在跨域
