webpack的使用

webpack-cli

npm 5.2以上版本中提供了npx 命令

npx解决的主要问题是调用项目内部安装的模块, 原理是在 node_modules 下的 .bin 目录中找到对应的命令执行

使用webpack命令: npx webpack

webpack4.0后可以实现零配置打包构建, 而零配置的特点是限制较多, 无法自定义很多配置

开发中常用的应是使用webpack配置进行打包构建

webpack配置

webpack四大核心概念:

  • 入门 entry : 打包入口目录

  • 输出 output : 打包生成目录

  • loader : 用于对模块的源代码进行转换

  • 插件 plugins : 插件用于解决 loader无法实现的其他问题

    1. 配置webpack.config.js

    2. 运行 npx webpack

    3. 将 npx webpack 命令配置到 package.json 的脚本中

      • 配置 package.json
      • 运行 npm run build

开发中自动编译工具

每次编译代码时, 手动运行 npm run build 是很繁琐的

webpack中有几个不同的选项, 可以帮助你在代码发生变化后自动编译代码:

  1. webpack’s Watch Mode
  2. Webpack-dev-server
  3. Webpack-dev-middleware

多数场景中, 可能需要使用 webpack-dev-sever ( 但不妨尝试以上的所有选项 )

watch

在 webpack 指令后添加 --watch 参数即可

主要作用于监视本地项目文件的变化, 侦查到代码变更便会自动编译打包生成输出文件

  1. 配置 package.json 的scripts

  2. 运行 npm run watch

以上是cli的方式设置watch参数

webpack-dev-server (推荐)

  1. 安装 devServer

    devServer 需要依赖 webpack , 所以在项目依赖中需安装 webpack

  2. 在 public 目录下创建 index.html

  3. 运行 npx webpack-dev-server

    • 在 package.json 中添加配置

    这是cli的方式设置devServer

    运行 npm run dev:cli

    • 亦可以在 webpack.config.js 中添加配置

    这是通过配置文件对devServer的参数进行修改

    运行 npm run dev

    devServer会在 内存中 生成打包好的 bundle.js 专供开发时使用 , 打包效率高, 修改代码后热更新, 用户体验非常好

html-webpack-plugin插件

  • devServer时根据模板在express项目根目录下生成html文件
  • devServer时自动引入bundle.js
  • 打包时自动生成index.html

  1. 安装html-webpack-plugin插件

  2. 在 webpack.config.js 中的 plugins 节点下配置

  3. 此时还应修改 public 目录下的 index.html

    html-webpack-plugin插件会自动引入打包好的 bundle.js 文件

webpack-dev-middleware

  • webpack-dev-middleware 是一个容器[wrapper], 它可以把webpack处理后的文件传递给一个服务器[server]
  • webpack-dev-server 在内部使用了它, 同时它也作为一个单独的包来使用, 以便进行更多自定义设置来实现更多的需求

如果使用 webpack-dev-middleware 必须使用 html-webpack-plugin 插件, 否则html文件无法正确输出到express服务器的根目录

  1. 安装 express 和 webpack-dev-middleware

  2. 新建 server.js

  3. 配置 package.json

  4. 运行 npm run server

####小结

  • 只有在开发时才需要使用自动编译工具 例如: webpack-dev-server
  • 项目上线时直接使用webpack进行打包构建 不需要使用自动编译工具
  • 自动编译工具只是为了提高开发体验

npm常用的命令

  • 全局安装 -g

React使用

Posted on 2019-01-20

React使用教程

  • React的生命周期
    • componentWillMount
      • 在渲染之前调用 客户端/服务端
    • componentDidMount
      • 在第一次渲染后调用 只在客户端 之后组件已经生成了对应的DOM结构 可以通过 this.getDOMNode() 来进行访问
      • 如果和其他 JavaScript框架一起使用 可以在此方法中调用 setTimeout()/ setInterval() 或者 Ajax请求等操作 (防止异步操作阻塞UI渲染)
    • componentWillReceiveProps
      • 在组件接收到新 props时调用
      • 此方法在初始化render时不会被调用
    • shouldComponentUpdate
      • 在组件接收到新的 props或者 state时调用
      • 在初始化时或者使用forceUpdate时不会被调用
      • 返回的是布尔值
      • 可在你确认不需要更新组件时使用
    • componentWillUpdate
      • 在组件接收到新的 props或者 state但还没有 render时调用
      • 在初始化时不会被调用
    • componentDidUpdate
      • 在组件完成更新后立即调用
      • 在初始化时不会被调用
    • componentWillUnmount
      • 在组件从DOM中移除的时候立即调用

Promise 介绍

  • 什么是 Promise

    • Promise 是异步编程的一种解决方案，专门处理回调地狱问题
    • Promise 是一个构造函数 其中有 resolve(成功回调) 和 reject(失败回调)
    • Promise 构造函数的 Prototype 属性中有 .then() 和 .catch() - - - [所有实例都能访问]
    • 实现链式编程
      • 由于 Promise 实例是异步操作 调用时会有两种状态
      • 状态 1 : 异步执行成功 内部调用成功回调函数 resolve() 返回结果
      • 状态 2 : 异步执行失败 内部调用失败回调函数 reject() 返回结果
      • 在创建 Promise 实例时调用 .then() 预先为异步操作指定 resolve(成功回调) 和 reject(失败回调)
        • 注意 [ .then() 必须有成功回调 可以没有失败回调 ]

  • 使用 Promise

  • 使用 Promise 解决回调地狱

    • 捕获异常的方式

      • // 读取多文件回调

        // 使用 .catch()方法 捕获其中任何 Promise执行失败的异常
