proxy代理

开发阶段往往会遇上 跨域 的问题, 但在上线后前端代码会和后端代码整合部署在 同一服务器 下, 并不会出现跨域的问题, 但是由于开发时用到 webpack-dev-server 所以一定会产生跨域的问题

目前跨域的主要解决方案有 :

  1. jsonp (淘汰)
  2. CORS
  3. http proxy

此处介绍的使用devServer解决跨域 原理是http proxy

将所有 ajax请求发送给devServer服务器, 再通过devServer服务器转发到数据接口服务器

因为跨域问题的起因是[浏览器的同源策略][https://developer.mozilla.org/en-US/docs/Web/Security/Same-origin_policy], 由于ajax请求是发送给devServer服务器, 所以不存在跨域

source map

在webpack配置中发现有时候我们需要查找报错代码在项目中的具体位置, 而webpack打包好的 .js 文件在引入时已经被处理过 此时我们可以通过设置 devtool来转换成 原始代码 (方便开发)

  • 开发环境推荐 cheap-module-eval-source-map

  • 生产环境推荐 none (不使用source map)

  • 原因如下:

    1. 使用 cheap模式可以大幅提高 source map 生成的效率。大部分情况我们调试并不关心列信息, 而且就算 source map没有列, 有些浏览器引擎 (例如 v8) 也会给出列信息。
    2. 使用 module 可支持 babel 这种预编译工具映射转换前的代码
    3. 使用 eval 方式可大幅提高持续构建效率。官方文档提供的速度对比表格可以看到 eval 模式的重新构建速度是很快的 [详细请看 [webpack官网][https://webpack.js.org/configuration/devtool/]]
    4. 使用 eval-source-map 模式可以减少网络请求。这种模式开启 DataUrl 本身包含完整的 source map信息, 并不需要像 sourceURL 那样, 浏览器需要发送完整请求去获取 source map 文件, 这会略微提高效率。而生产环境中则不宜用 eval, 这样会让文件变得极大。

  • 设置 webpack.config.js

React使用教程

  • React的生命周期
    • componentWillMount
      • 在渲染之前调用 客户端/服务端
    • componentDidMount
      • 在第一次渲染后调用 只在客户端 之后组件已经生成了对应的DOM结构 可以通过 this.getDOMNode() 来进行访问
      • 如果和其他 JavaScript框架一起使用 可以在此方法中调用 setTimeout()/ setInterval() 或者 Ajax请求等操作 (防止异步操作阻塞UI渲染)
    • componentWillReceiveProps
      • 在组件接收到新 props时调用
      • 此方法在初始化render时不会被调用
    • shouldComponentUpdate
      • 在组件接收到新的 props或者 state时调用
      • 在初始化时或者使用forceUpdate时不会被调用
      • 返回的是布尔值
      • 可在你确认不需要更新组件时使用
    • componentWillUpdate
      • 在组件接收到新的 props或者 state但还没有 render时调用
      • 在初始化时不会被调用
    • componentDidUpdate
      • 在组件完成更新后立即调用
      • 在初始化时不会被调用
    • componentWillUnmount
      • 在组件从DOM中移除的时候立即调用

Promise 介绍

什么是 Promise

  • Promise 是异步编程的一种解决方案，专门处理回调地狱问题
  • Promise 是一个构造函数 其中有 resolve(成功回调) 和 reject(失败回调)
  • Promise 构造函数的 Prototype 属性中有 .then() 和 .catch() - - - [所有实例都能访问]

  • 实现链式编程

    • 由于 Promise 实例是异步操作 调用时会有两种状态
    • 状态 1 : 异步执行成功 内部调用成功回调函数 resolve() 返回结果
    • 状态 2 : 异步执行失败 内部调用失败回调函数 reject() 返回结果

    • 在创建 Promise 实例时调用 .then() 预先为异步操作指定 resolve(成功回调) 和 reject(失败回调)

      注意: [ .then() 必须有成功回调 可以没有失败回调 ]
