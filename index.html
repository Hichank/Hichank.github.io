npm常用命令

Posted on 2019-01-20

npm常用的命令

  • 全局安装 -g

    • 全局安装的包位于Node.js环境的node_modules目录下
    • 全局安装的包一般用于命令行工具
    • npm install -g 包名称

  • 本地安装

    • 本地安装的包在当前目录下的node_modules里面
    • 本地安装的包一般用于实际的开发工作
    • npm install 包名称
    安装包的时候可以指定版本
    • npm install -g 包名称@版本号
    卸载包
    • npm uninstall -g 包名称
    更新包 ( 更新到最新版本 )
    • npm update -g 包名称
    • npm install 包名称@latest (直接覆盖旧版本)
    开发环境
    • 平时开发使用的环境
    生产环境

    • 项目部署上线之后的服务器环境

      • – save (向生产环境添加依赖 dependencies)

      • 简写 -S

      • – save-dev (向开发环境添加依赖 devDependencies)

      • 简写 -D

    • 环境依赖切换 npm install –production (只安装生产环境)

    • 直接 npm install (所有环境都安装)

yarn工具的基本使用

  • 安装yarn工具 : npm install -g yarn
    • 初始化包
      • npm init
      • yarn init
    • 安装包
      • npm install xxx –save
      • yarn add xxx
    • 移除包
      • npm uninstall xxx
      • yarn remove xxx
    • 更新包
      • npm update xxx
      • yarn upgrade xxx
    • 安装开发依赖包
      • npm install xxx –save-dev
      • yarn add xxx –dev
    • 全局安装
      • npm install -g xxx
      • yarn global add xxx
    • 设置下载镜像地址
      • npm config set registry url
      • yarn config set registry url
    • 安装所有依赖
      • npm install
      • yarn install
    • 执行包
      • npm run
      • yarn run

解决npm安装包被墙的问题

  • –registry
  • cnpm
    • 淘宝npm镜像 与官方NPM的同步频率目前为10分钟一次
      • npm config get registry 查看 registry
      • npm config set registry 修改 registry
    • 官网: http://npm.taobao.org/
    • npm install -g cnpm -registry=https//registry.npm.taobao.org
    • 使用cnpm安装包 cnpm install 包名称
  • nrm
    • 作用: 修改镜像源
    • 项目地址: https://www.npmjs.com/package/nrm
    • 安装: npm install -g nrm

React使用

Posted on 2019-01-20

React使用教程

  • React的生命周期
    • componentWillMount
      • 在渲染之前调用 客户端/服务端
    • componentDidMount
      • 在第一次渲染后调用 只在客户端 之后组件已经生成了对应的DOM结构 可以通过 this.getDOMNode() 来进行访问
      • 如果和其他 JavaScript框架一起使用 可以在此方法中调用 setTimeout()/ setInterval() 或者 Ajax请求等操作 (防止异步操作阻塞UI渲染)
    • componentWillReceiveProps
      • 在组件接收到新 props时调用
      • 此方法在初始化render时不会被调用
    • shouldComponentUpdate
      • 在组件接收到新的 props或者 state时调用
      • 在初始化时或者使用forceUpdate时不会被调用
      • 返回的是布尔值
      • 可在你确认不需要更新组件时使用
    • componentWillUpdate
      • 在组件接收到新的 props或者 state但还没有 render时调用
      • 在初始化时不会被调用
    • componentDidUpdate
      • 在组件完成更新后立即调用
      • 在初始化时不会被调用
    • componentWillUnmount
      • 在组件从DOM中移除的时候立即调用

Promise 介绍

  • 什么是 Promise

    • Promise 是异步编程的一种解决方案，专门处理回调地狱问题
    • Promise 是一个构造函数 其中有 resolve(成功回调) 和 reject(失败回调)
    • Promise 构造函数的 Prototype 属性中有 .then() 和 .catch() - - - [所有实例都能访问]
    • 实现链式编程
      • 由于 Promise 实例是异步操作 调用时会有两种状态
      • 状态 1 : 异步执行成功 内部调用成功回调函数 resolve() 返回结果
      • 状态 2 : 异步执行失败 内部调用失败回调函数 reject() 返回结果
      • 在创建 Promise 实例时调用 .then() 预先为异步操作指定 resolve(成功回调) 和 reject(失败回调)
        • 注意 [ .then() 必须有成功回调 可以没有失败回调 ]

  • 使用 Promise

  • 使用 Promise 解决回调地狱
